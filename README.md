# CatThief

Высоконагруженный TypeScript-скрипт для массового получения изображений с удалённого API, дедупликации полученных файлов и отправки уникальных изображений обратно на сервер в ZIP-архивах.

## О проекте

**CatThief** запускает множество параллельных воркеров, которые:

1. выполняют `GET` запросы к API,
2. получают бинарные данные изображения,
3. фильтруют дубликаты по хэшу,
4. накапливают батч уникальных картинок,
5. собирают ZIP-архив,
6. отправляют архив `POST`-запросом на тот же endpoint.

---

## Технологии

- **TypeScript** (строгий режим)
- **Node.js**
- **undici** (HTTP-клиент и пул соединений)
- **JSZip** (создание ZIP-архивов)

## Конфигурация (в коде)

Ключевые параметры находятся в начале `cat-thief-plus.ts`:

- `CAT_URL` — целевой API endpoint.
- `CONCURRENCY` — количество параллельных воркеров.
- `BATCH_SIZE` — сколько уникальных изображений собирать в один ZIP.
- `MAX_CONNECTIONS` — лимит одновременных HTTP-соединений.

При необходимости значения можно скорректировать прямо в файле перед запуском.

## Подготовка окружения

### 1) Требования

- **Node.js 20+** (рекомендуется LTS)
- **npm 10+**
- доступ в интернет к `http://algisothal.ru:8889`

Проверка версий:

```bash
node -v
npm -v
```

### 2) Установка зависимостей

В репозитории нет `package.json`, поэтому сначала инициализируйте проект:

```bash
npm init -y
```

Установите runtime-зависимости:

```bash
npm install jszip undici
```

Установите dev-зависимости для TypeScript:

```bash
npm install -D typescript @types/node
```

## Что выводится в логах

Во время работы скрипт печатает:

- стартовые параметры (URL, число воркеров, лимиты),
- прогресс по уникальным изображениям и общему числу запросов,
- количество созданных и отправленных архивов,
- итоговую статистику при корректном завершении.

Остановка — через `Ctrl + C` (SIGINT): скрипт выводит финальную сводку перед завершением.

## Как это работает

- Для каждого запроса вызывается `fetchCat()`.
- Уникальность определяется в `hashImage()` по размеру буфера и первым 64 байтам.
- Уникальные изображения накапливаются в `currentBatch`.
- Когда батч достигает `BATCH_SIZE`, вызываются `buildZip()` и `sendArchive()`.
- Отправка архива выполняется асинхронно и не блокирует основной цикл воркеров.

## Ограничения и нюансы

- Проект ориентирован на очень высокую конкуррентность (по умолчанию 3000 воркеров и 4000 соединений, на тестых разгонялось до 50000), что может сильно нагружать сеть, CPU и удалённый сервис.
- Дедупликация реализована эвристически (не криптографический хэш) — это быстрее, но теоретически возможны коллизии.
- Ошибки сетевых операций в текущей версии подавляются (`catch {}`), что полезно для непрерывного стресс-режима, но снижает наблюдаемость проблем.
